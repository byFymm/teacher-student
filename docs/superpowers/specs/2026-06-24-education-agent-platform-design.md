# 教育智能体平台设计规格

日期：2026-06-24

## 1. 背景

本项目是在 `D:\TEACHER` 下新建的教育智能体平台。平台面向多机构 SaaS 场景，第一期同时服务老师和学生：老师可以上传教学资料和题库，创建课程智能体和考试训练任务；学生可以围绕课程资料进行问答、刷题、考试训练和错题复习；系统把学习过程沉淀为学情数据反馈给老师。

毕昇工作流平台作为核心智能体编排引擎参与第一期。平台自身负责教育业务产品化和多租户数据治理，毕昇负责资料解析、知识库问答、出题组卷、批改讲解和学情报告等智能流程。

## 2. 已确认决策

- 第一优先服务对象：学生 + 老师。
- 第一教学场景：通用课堂助手 + 考试/刷题训练。
- 毕昇定位：核心编排引擎。
- 技术栈：Python/FastAPI 后端 + React 或 Vue 前端。
- 资料范围：文档资料 + 结构化题库。
- 模型接入：优先使用毕昇已配置模型，同时支持国产模型供应商可配置。
- 平台形态：多机构 SaaS。
- 毕昇部署：纳入开发计划，从零部署并联调。
- 总体方案：教育业务平台 + 毕昇核心编排，通过适配层隔离。

## 3. 目标

第一期目标是跑通一个完整教学闭环：

1. 管理员创建机构、用户、角色、模型供应商配置和毕昇工作流绑定。
2. 老师创建课程和班级，上传 PDF、Word、PPT、TXT 等文档资料。
3. 老师导入 Excel、CSV 或 JSON 题库，并关联课程与知识点。
4. 平台触发毕昇完成资料解析、知识库入库、出题组卷、批改讲解和报告生成。
5. 学生进入课程后进行课程问答、章节练习、考试训练和错题复习。
6. 老师查看学生答题记录、薄弱知识点、错题分布和班级学情。

## 4. 非目标

第一期不做以下内容：

- 不做视频或音频课程解析链路。
- 不做复杂教务排课系统。
- 不做支付、计费、套餐和商业化订阅。
- 不做跨机构资源市场。
- 不做完整低代码工作流设计器；毕昇自身承担工作流配置能力。
- 不让前端直接依赖毕昇 API。

## 5. 用户角色

### 学生

学生围绕课程学习。核心操作包括课程问答、练习刷题、考试训练、查看答案讲解、查看错题本和复习建议。

### 老师

老师围绕课程教学。核心操作包括创建课程、管理班级、上传资料、导入题库、生成练习或试卷、发布任务、查看学情报告。

### 机构管理员

机构管理员负责平台运行配置。第一期只做轻量能力，包括机构信息、用户账号、角色权限、模型供应商配置和毕昇工作流绑定。

## 6. 总体架构

系统分为四层：

1. 用户层：学生端、教师端、机构管理端。
2. 教育业务后端：FastAPI，负责多租户、权限、课程、班级、资料、题库、学习记录、学情报告和稳定 API。
3. 工作流适配层：负责把教育业务事件转换为毕昇工作流调用，处理任务状态、回调、重试和结果归档。
4. 智能与基础设施层：毕昇工作流平台、PostgreSQL、对象存储、向量库、Redis/任务队列、国产模型供应商。

核心原则是：平台管理教育业务数据，毕昇管理智能流程能力。课程、班级、学生、题库、答题记录等业务数据不交给毕昇作为主数据；平台只保存毕昇的 `workflow_id`、`run_id`、知识库引用、执行状态和结果摘要。

## 7. 前端信息架构

第一期登录后按角色进入工作台，不做营销首页。

### 学生学习工作台

主要页面：

- 我的课程
- 课程详情
- AI 问答
- 章节练习
- 考试训练
- 错题本
- 学习报告

### 教师教学工作台

主要页面：

- 课程管理
- 班级管理
- 资料上传与解析状态
- 题库导入与维护
- 试卷/练习生成与发布
- 班级学情看板

### 机构管理工作台

主要页面：

- 租户/机构信息
- 用户与角色
- 模型供应商配置
- 毕昇工作流绑定
- 系统运行日志

## 8. 后端核心模块

### 租户与权限模块

核心对象：

- `Tenant`
- `School`
- `User`
- `Role`
- `Class`
- `Enrollment`

所有业务表带 `tenant_id`。访问课程、班级、资料、题库、学习记录和报告时都必须校验租户和角色。

### 课程与资源模块

核心对象：

- `Course`
- `TeachingResource`
- `ResourceParseJob`

课程是一期业务中心。课程关联老师、班级、资料、题库、知识库引用、智能体配置和考试任务。

### 题库与测评模块

核心对象：

- `QuestionBank`
- `Question`
- `QuestionOption`
- `ExamPaper`
- `ExamSession`

题库支持结构化导入。客观题由平台本地判分，主观题和开放题由毕昇工作流辅助批改与讲解。

### 智能体与工作流模块

核心对象：

- `AgentTemplate`
- `BishengWorkflowBinding`
- `WorkflowRun`
- `CallbackEvent`

该模块提供统一适配层，不向前端暴露毕昇内部接口。

### 学习过程模块

核心对象：

- `Conversation`
- `Message`
- `PracticeSession`
- `AnswerRecord`
- `Mistake`

学习过程数据用于错题本、掌握度分析和学情报告。

### 学情分析模块

核心对象：

- `KnowledgePointMastery`
- `StudentReport`
- `ClassAnalytics`

第一期以统计和模型辅助总结为主，不做复杂学习路径算法。

## 9. 毕昇集成设计

### 适配层契约

平台后端定义统一适配层：

- `start_workflow(workflow_key, payload)`
- `get_run_status(run_id)`
- `stream_chat(workflow_key, payload)`
- `handle_callback(callback_payload)`

具体毕昇 API 路径、认证方式和请求格式在毕昇部署完成后由真实适配器实现。业务层只依赖适配层接口。

### 核心工作流

#### 资料入库工作流

老师上传文件后，平台保存文件和解析任务，调用毕昇进行资料解析和知识库入库。毕昇完成后回写知识库引用、解析状态、错误信息和可检索范围。

#### 课程问答工作流

学生在课程内提问。平台先校验学生是否属于课程或班级，再调用毕昇 RAG 问答工作流。答案可以流式返回，最终保存到对话记录。

#### 出题组卷工作流

老师选择课程、章节、知识点、难度和题型后，平台创建组卷任务，调用毕昇基于资料和题库生成试卷。平台保存生成后的试卷和题目，老师确认后发布。

#### 批改报告工作流

学生提交答案后，平台本地判定客观题；主观题调用毕昇进行评分、点评和讲解。系统生成答题记录、错题、掌握度和学情报告。

## 10. 错误处理

所有长流程都按任务处理，状态包括：

- `pending`
- `running`
- `succeeded`
- `failed`
- `retryable`

资料解析、组卷、报告生成等任务失败时，平台保留错误原因、毕昇运行编号、请求摘要和重试次数。老师可以重试，不删除原始资料和课程数据。

课程问答失败时，前端展示明确错误提示，并允许学生重新提问。平台保留失败消息，便于排查。

回调处理必须幂等。重复回调只更新同一个 `WorkflowRun`，不重复创建报告或题目。

## 11. 安全与权限

- 所有核心业务表带 `tenant_id`。
- 学生不能直接访问题库答案。
- 学生只能访问自己所在班级或被授权课程。
- 老师只能管理自己机构或班级范围内的课程、资料、题库和报告。
- 机构管理员只能管理本机构配置。
- 模型密钥不返回前端。
- 文件上传限制类型、大小和租户配额。
- AI 输出必须保存来源任务和使用场景，方便审计。
- 出题、批改、报告等影响教学判断的结果，老师端可以预览、编辑和确认。

## 12. 测试策略

### 后端单元测试

覆盖：

- 多租户隔离
- 角色权限
- 课程、资料、题库服务
- 工作流适配层请求构造
- 回调解析和幂等处理

### 集成测试

先使用假毕昇适配器跑通四条流程：

- 资料入库
- 课程问答
- 出题组卷
- 批改报告

毕昇部署完成后，再切真实适配器进行联调测试。

### 前端 E2E 测试

覆盖三条主路径：

- 老师创建课程、上传资料、导入题库。
- 学生进入课程、问答、练习、考试。
- 老师查看学生错题和班级学情。

## 13. 验收标准

第一期完成后应能演示：

1. 管理员创建机构、用户和角色。
2. 老师创建课程和班级，并上传文档资料。
3. 老师导入题库，生成并发布一次练习或考试。
4. 学生进入课程完成一次问答和一次练习。
5. 系统生成答案讲解、错题记录和学情统计。
6. 老师查看班级学情报告。
7. 毕昇工作流调用失败时，平台能展示状态并支持重试。

## 14. 开发顺序

建议按以下顺序开发：

1. 项目脚手架、数据库、认证、多租户和权限底座。
2. 课程、班级、资料、题库基础 CRUD。
3. 毕昇部署、模型供应商配置和工作流适配层。
4. 资料入库工作流和课程问答闭环。
5. 题库导入、出题组卷和考试训练闭环。
6. 批改讲解、错题本和学情报告。
7. 教师端、学生端、管理端前端体验打磨。
8. 集成测试、E2E 测试和部署文档。

## 15. 关键风险

- 毕昇真实 API、认证方式和部署形态需要在部署后验证。适配层用于隔离该风险。
- 多租户安全必须从建表和查询层同时落实，否则后期补救成本很高。
- 文档解析和题库导入的格式差异较大，第一期应先限制模板和文件大小。
- 模型输出不稳定，出题、批改和报告必须支持老师确认和编辑。
- 学情分析第一期应保持可解释，不做复杂黑盒评分。
