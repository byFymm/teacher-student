# 教育智能体平台 MVP 功能闭环设计

日期：2026-06-24

## 背景

当前仓库已经具备 FastAPI 后端、React 前端、三角色工作台和假毕昇适配器骨架。本轮目标不是一次性完成完整 SaaS，而是把静态界面推进到可演示的第一条业务闭环：老师创建教学资料和练习，平台模拟触发 Bisheng 工作流，学生收到学习任务并能进行 AI 问答，管理员能维护模型/工作流接入配置。

## 本轮目标

1. 后端提供内存态业务数据和稳定 API，支撑演示闭环。
2. 教师端可以新增资料解析任务、生成练习任务，并看到工作流状态刷新。
3. 学生端可以查看课程、任务、知识点掌握度，向 AI 学习助手提问，并完成任务状态变更。
4. 管理端可以新增供应商配置、启停供应商，并看到审计日志。
5. 继续保持三端页面的高级感和中文体验，不引入复杂登录、数据库迁移或真实文件上传。

## 非目标

- 本轮不接真实 Bisheng HTTP API。
- 本轮不接 PostgreSQL、MinIO、Redis 或 Celery。
- 本轮不做真实认证、租户权限和文件二进制上传。
- 本轮不做完整考试作答页，只做任务产生与学习助手闭环。

## 后端设计

新增一个内存业务仓库，集中保存课程运营、工作流运行、资料解析、学生任务、AI 对话、供应商配置和审计日志。仓库由 FastAPI 进程持有，适合本地演示和测试；后续接数据库时可以保留 API 契约，替换服务实现。

新增 API：

- `GET /api/v1/teacher/workspace`：返回教师工作台数据。
- `POST /api/v1/resources`：创建资料解析任务，并追加一个资料入库工作流。
- `POST /api/v1/exercises`：创建练习生成任务，并追加学生任务和工作流。
- `GET /api/v1/student/workspace`：返回学生工作台数据。
- `POST /api/v1/chat`：模拟课程知识库问答，保存会话消息。
- `PATCH /api/v1/student/tasks/{task_id}`：更新学生任务状态。
- `GET /api/v1/admin/workspace`：返回管理端配置与日志。
- `POST /api/v1/admin/providers`：新增模型或工作流供应商。
- `PATCH /api/v1/admin/providers/{provider_id}/toggle`：启停供应商。

## 前端设计

前端继续使用现有三端工作台布局，不重新设计页面结构。新增 TanStack Query 数据请求、mutation 和成功/失败状态。

教师端：

- 统计卡、课程运营、资料、工作流、洞察数据从 API 获取。
- “上传资料”区域改成可输入资料名称和类型的表单。
- “练习生成器”提交后创建练习任务，并刷新工作流和学生任务。

学生端：

- 课程、任务、掌握度、问答记录从 API 获取。
- AI 学习助手提供输入框，提交问题后显示回答。
- 今日任务支持点击切换为“已完成”，刷新统计。

管理端：

- 供应商状态、系统模块、审计日志从 API 获取。
- 新增供应商表单支持配置名称、类型和延迟。
- 每个供应商提供启停按钮，并写入审计日志。

## 错误与状态

所有写操作在前端展示轻量状态提示。后端对空名称、空知识点、空问题、无效任务或供应商 ID 返回 400/404。由于本轮是内存演示，页面刷新后仍保留进程内数据；服务重启会回到种子数据。

## 测试与验收

后端测试覆盖：

- 创建资料会新增资料记录和工作流。
- 创建练习会新增工作流和学生任务。
- 学生问答会返回助手答案并累计会话。
- 学生任务状态可以更新。
- 管理员可以新增并启停供应商。

前端验收：

- 教师端能提交资料和练习表单。
- 学生端能发送问题并看到回答。
- 管理端能新增供应商并切换状态。
- 桌面和移动视口无空白、无控制台错误、无横向溢出。
